Homework 1: Tasks for sd545

Your personal task is to develop a system for depots and their buses.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Bus. With the following private fields:

  1. String manufacturer
  2. String driver
  3. String city
  4. int route
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Depot. With the following private fields:

  1. int id
  2. String location
  3. String owner
  4. List<Bus> buses
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Depot> depots in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one depot with two buses. [4 marks]

Create controller co2103.hw1.controller.DepotController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/depots" should show view depots/list. [2 marks]
    • Add the list of all depots from co2103.hw1.Hw1Application to the model as attribute depots. [2 marks]
    • Create the corresponding JSP to list the location and owner of each depot followed by a link to URL "/buses?depot={id of depot}", e.g., "/buses?depot=3". [4 marks]
  2. Requests for URL "/newDepot" should show view depots/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="depot" and action="/addDepot" and inputs for all fields of class Depot except for buses. [5 marks]
  3. POST requests for URL "/addDepot" should add a new depot to the list of all depots in co2103.hw1.Hw1Application and redirect to /depots. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.BusController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/buses" should show view buses/list. [2 marks]
    • Based on request parameter depot determine the corresponding depot from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all buses of that depot to the model as attribute buses. [2 marks]
    • Create the corresponding JSP for the view to list the manufacturer, driver, city, and route of each bus. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newBus?depot={id of depot whose buses are listed}", e.g., "/newBus?depot=3". [4 marks]
  2. Requests for URL "/newBus" should show view buses/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="bus" and action="/addBus" and inputs for all fields of class Bus. [6 marks]
    • Also add the request parameter depot with the ID of the current depot. [4 marks]
  3. POST requests for URL "/addBus" should add a new bus to the list of buses of the depot identified by request parameter depot and then redirect to /depots. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Depot and make it available in controller DepotController. [4 marks]

When adding a new depot validate that

  1. The value of field id is unique within the list Hw1Application.depots. [3 marks]
  2. The value of field location is not empty. [2 marks]
  3. The value of field owner is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Bus and make it available in controller BusController. [4 marks]

When adding a new bus validate that

  1. The value of field manufacturer is not empty. [2 marks]
  2. The value of field driver is not empty. [2 marks]
  3. The value of field city is either "Leicester", "Birmingham", or "Nottingham". [3 marks]
  4. The value of field route is between (including) 1 and 250. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
